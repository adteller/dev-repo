- где располагается репозиторий с кодом, как посмотреть существующий код:
https://github.com/adteller/dev-repo

- как устроен процесс внесения своих изменений в основную кодовую базу:
Создается копия рабочей ветки.
в неё вносятся изменения.
после чего создается pull request на с лияние с основной веткой.
После проверки кода, код отправляется на дороботку либо сливается с основной веткой.

- правила именования веток:
master - Продакшн релиз
develop - Ветка разработки
release_N - Версионность продукта
bugfix - Ветка исправления ошибок

- Конфигурационный фал Git
https://github.com/adteller/dev-repo/blob/master/config


- инструкция для младших сотрудников по слиянию веток, какие команды надо выполнить, чтобы:

- создать свою ветку: git branch "name branche"

- внести изменения в нее: 
Переключится на ветку - git checkout "name branche"
После внесения изменений: git status
git add .
git commit -m "commit"
git push origin "name branche"

- слить эти изменения с основной кодовой базой. 
Перейти в сливаемую ветку: git checkout "name branche"
выполнить команду: git merge "Имя ветки откуда сливаем"
